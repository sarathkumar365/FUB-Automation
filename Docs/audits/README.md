# Audits

Point-in-time reviews of the codebase — stale code, dependency health, architecture drift, etc.

## Conventions
- File name: `<topic>-audit-YYYY-MM-DD.md` (date the audit was run).
- Each audit is a snapshot; do not edit old audits in place — write a new one.
- If an audit produces follow-up work, open issues or feature docs and link them from the audit.
