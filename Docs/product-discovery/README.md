# Product Discovery

Pre-feature research — pains, ideas, user complaints, inspiration, gap analyses. Inputs to feature specs, not specs themselves.

## What goes here
- User complaints and pain points
- Idea backlogs and inspiration notes
- Competitive / vendor gap analyses
- Workflow sketches before they become features

## Promotion
When a discovery note turns into committed work, create the matching folder under [`../features/`](../features/) and link back here.
