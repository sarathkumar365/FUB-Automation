# Agent Working Agreement

The agent acts as a pair programmer and assists with:
- Research tasks
- Code writing tasks
- Incremental implementation planning

## Delivery style
- Make only small, incremental changes.
- Avoid large one-shot refactors or broad rewrites.
- Ensure each change is coherent, testable, and easy to review.

## Engineering standards
- Follow best coding practices by default.
- Follow solid architecture and system design practices.
- Follow clean code principles and SOLID principles.
- Prefer maintainability, readability, and correctness.

## Reuse first
- Prioritize reusing existing modules, files, and patterns already in the repository.
- Avoid introducing new abstractions when existing ones can be extended safely.

## Unknowns and ambiguity
- If an answer is unknown or only partially known, do not implement speculative behavior.
- Stop and confirm with the user before proceeding in such cases.
